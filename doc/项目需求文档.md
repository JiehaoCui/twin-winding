## 一、系统概述

控制板控制3个外部步进电机驱动器（共用内部485接口）+ 1个板载步进电机驱动器，实现丝线自动卷绕、张力闭环控制、自动换卷、参数统计上报等功能。车头PLC通过外部485下发参数和接收状态。

### 核心功能

1. **卷绕电机控制**（2个工位：A/B） - 张力闭环 + 换卷切换
2. **排线电机控制**（1个） - 跟随卷绕速度，往复运动
3. **换卷步进电机**（板载） - 驱动机械结构实现换卷
4. **按钮面板交互** - I2C IO扩展芯片，10个按键 + 急停 + LED
5. **参数监控上报** - 与车头PLC通过外部485通信

## 三、系统工作流程

### 3.1 上电初始化

1. **通信检测**: 确认3个外部驱动器是否在线（通过内部485）
2. **排线电机回零**: 控制排线电机回到零位（机械原点）
3. **工位选择**: 等待车头PLC下发当前工位（A或B）

### 3.2 等待启动

- 显示系统状态（版本、电机转速、产量、故障等）
- 等待启动按键触发或车头PLC启动指令

### 3.3 运行控制

#### 3.3.1 张力闭环控制

**控制逻辑**:

```
参考目标转速 = 放丝线速度 / (π × 实测卷绕直径 × 卷绕减速比)
实际输出转速 = 参考目标转速 × (1 + 张力环输出)
```

其中，张力环输出范围为 **[-1.1, 0.5]**：
- 张力环输出 = 0 → 实际输出 = 参考目标转速（摆臂平衡）
- 张力环输出 = -1.1 → 实际输出 = 0（摆臂过低，停止）
- 张力环输出 = 0.5 → 实际输出 = 1.5 × 参考目标转速（摆臂过高，加速）

**张力PID控制**:

```
摆臂位置（模拟输入） → 误差 = 目标位置 - 实际位置 → 张力PID → 张力环输出
```

- **目标位置**: 摆臂中间位置（张力平衡）
- **摆臂位置**: 模拟采集值（0~VCC，中间值 = 平衡点）
- **控制方式**: 脉冲宽度 = 实际输出转速对应的频率
- **方向**: 由实际输出转速的正负决定
- **使能**: 通过内部485发送到当前工位卷绕驱动器

**PID参数**: 由车头PLC下发（Kp、Ki）
**特点**: 此控制方式（基准转速 + 张力调节）抖动较小，适合丝线张力控制

#### 3.3.2 排线跟随控制

```
卷绕锭子转速 × 卷绕减速比 = 卷绕电机转速
卷绕电机转速 × 线间距 × 排线减速比 = 排线电机转速
排线电机转速 → 脉冲频率输出（定时器实现）
排线方向 → 方向口输出
```

- **卷绕减速比**: 车头PLC下发（卷绕锭子转速 : 卷绕电机转速）
- **线间距**: 车头PLC下发（卷绕转一圈，排线行走距离）
- **排线减速比**: 车头PLC下发（排线电机角度 : 机械行走距离）
- **目标**: 卷绕转一圈，排线行走距离 = 线间距

#### 3.3.3 卷绕直径计算与米长统计

**卷绕直径计算**:

```
卷绕锭子转速 = 卷绕电机转速 × 卷绕减速比
放丝线速度 = 卷绕锭子转速 × π × 卷绕直径
∴ 卷绕直径 = 放丝线速度 / (π × 卷绕锭子转速)
```

**注意**: 放丝线速度由车头PLC实时下发，卷绕锭子转速由电机编码器或脉冲反馈获取。

**米长统计**:

```
米长 = ∑ (放丝线速度 × Δt)
```

- **放丝线速度**: 车头PLC实时下发
- **满卷判断**:
  - 实测卷绕直径 > 车头PLC下发的满卷直径
  - 或 实测米长 > 车头PLC下发的满卷米长
  - 任一条件满足即判定满卷

#### 3.3.4 排线动程控制（往复运动）

```
空管动程（PLC下发） → 随直径增加逐步缩短两边 → 形成收边角度
```

- **空管动程**: 车头PLC下发（空管时排线往复行程）
- **收边角度**: PLC下发（随直径增加，两边缩短的角度）
- **控制**: 排线在特定角度范围内往复运动

### 3.4 自动换卷流程

**触发条件**: 满卷（直径或米长超标）

**换卷动作序列**:

1. **启动待机工位**: 另一个工位的卷绕电机旋转，线速度 = 放丝线速度 / (π × 空管直径)
2. **张力保持**: 张力跟踪仍由当前工位实现
3. **步进电机换卷**:
   - 正转/反转 → 机械结构移动 → 纱线碰到新工位卷绕锭子
   - 步进电机旋转到预定位置
4. **张力切换**: 张力跟踪切换到新工位实现
5. **步进电机复位**: 反方向旋转回原位
6. **原工位停止**: 之前工作的卷绕电机停止

## 5.A 换卷张力切换策略（已确认）

### 场景描述

满卷后，当前工位（如A）卷绕电机持续工作，启动待机工位（B）电机空转，步进电机换卷后，张力跟踪从A切换到B。

### 采用方案：速度匹配 + 瞬时切换

**实现流程**:

```
1. 步进电机换卷到位后，暂停换卷动作（步进电机停止）
2. B电机在空管下加速到当前线速度对应的转速
   V_B = 放丝线速度 / (π × 空管直径)
3. A电机继续张力控制，确保摆臂平衡
4. 监测B电机实际速度，当 |V_B - V_目标| < ε 时：
   - 瞬时将张力PID输出切换到B电机
   - A电机停止
5. 步进电机复位
```

**关键参数**:
- `T_match`（速度匹配超时时间）: 500ms~1s，超时则强制切换
- `ε`（速度匹配阈值）: ±5%以内即认为匹配成功
- `T_switch`（切换后张力稳定时间）: 200ms，确保PID收敛后再继续正常卷绕

**状态机设计**:

```
状态: FULL_WINDING（满卷）
  → 换卷启动
    → B电机加速到目标速度（速度匹配状态）
    → 检测速度差 |V_B - V_目标| < ε?
      YES: 张力切换到B电机
      NO:  超时T_match，强制切换
    → A电机停止
    → 步进电机复位
    → 状态: RUNNING（恢复卷绕）
```

---

### 优势

1. **切换瞬间两个电机转速基本一致**，张力跳变小
2. **切换过程可控**，可通过监测速度差确保安全
3. **实现复杂度适中**（比渐变切换简单，比硬切换更可靠）
4. **有硬件支持**：驱动器将电机编码器信号接过来，具备速度监测条件

## 六、初步技术选型

### 6.1 MCU选型

- **STM32F407VET6**: 已确认
- **资源需求**:
  - UART: 至少3个
    - UART1: 内部485（3个电机驱动器）
    - UART2: 外部485（车头PLC，Modbus RTU）
    - UART3: 调试串口
  - TIM: 至少4个独立TIM用于脉冲输出
    - TIM1: 卷绕电机A脉冲输出
    - TIM2: 卷绕电机B脉冲输出
    - TIM3: 排线电机脉冲输出
    - TIM4: 换卷步进电机脉冲输出
  - ADC: 2路（摆臂张力 + 母线电压）
  - I2C: 1路（HTR3335 IO扩展）
  - GPIO: 若干（方向口、限位开关、急停等）

### 6.2 RTOS

- **FreeRTOS**: 成熟稳定，优先考虑
- **任务划分**:
  - 主任务（状态机）
  - 张力控制任务（PID）
  - 排线控制任务（往复运动）
  - 通信任务（485与PLC交互）
  - 按键扫描任务（I2C IO）
  - 监控任务（电压/故障检测）

### 6.3 软件模块划分

```
├── Driver (硬件驱动)
│   ├── GPIO/ADC/DMA
│   ├── TIM (脉冲输出)
│   ├── UART (485通信)
│   └── I2C (IO扩展)

│   ├── 步进电机驱动接口
```
